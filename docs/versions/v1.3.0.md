# v1.3.0 - 2025-12-20

**Status:** RELEASED

## Summary

Complete TypeScript migration for both frontend and backend codebases.

---

## Changes

### Changed
- **Full TypeScript Migration:**
  - Frontend: All `.jsx` files converted to `.tsx`
  - Backend: All `.js` files converted to `.ts`
  - Shared type definitions for cross-codebase consistency
  - TypeScript configuration: `tsconfig.json`, `tsconfig.base.json`

### Fixed
- Docker build configuration for TypeScript compilation
- ES module default export interop for dynamic `require()` calls
- Frontend path resolution in production Docker container
- Log buffer integration in compiled TypeScript backend

---

## Technical Details

**TypeScript Configuration:**
- `tsconfig.base.json`: Shared compiler options
- `tsconfig.json`: Frontend IDE support
- `server/tsconfig.json`: Backend compilation with CommonJS output

**Docker Changes:**
- Added TypeScript compilation step in Dockerfile
- Updated CMD to use compiled JS path (`server/dist/server/index.js`)
- Copies `tsconfig.base.json` and `shared/` for compilation

**ES Module Interop:**
- Dynamic `require()` calls now use `.default` fallback for ES module exports
- Affected files: `notifications.ts`, `logger.ts`
